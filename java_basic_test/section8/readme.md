## final

최초 한번만 할당 가능한 것.

```java
final int number;
number = 1;
number = 2; ( x )
혹은

final int number = 1;
number = 2; ( x )
두 변수 모두 값이 할당되면 다시는 못바꿈
```

말 그대로 "끝"! 조금더 와닿게 표현하자면 "마감" ? 이라고 생각하면 된다.

절대 값을 바꿀수 없음..!!

method 에 들어가는 파라미터 값도 final 로 선언 가능하지만 그 안에 있는 값은 절대 바꿀수없다

```java
..method(final int number){

}
이런식으로 가능하다 여기서 number 은 메서드 사용시 파라미터 값을 넣어준 값에서 절때 바꿀수없다 컴파일러 오류가 발생함
```

* final + 필드 초기화를 사용할경우 static final을 붙여서 사용하는것이 효과적이다. ( 상수 )

* 상수란 ?

변하지 않고 항상 일정한 값을 갖는 수 

자바에서는 단 하나만 존재하는 변하지 않는 고정된 값을 상수라 칭한다

특징은 대문자를 사용하며 구분은 _ (언더스코어) 를 사용한다.
일반적인 변수와 상수를 구분하기 위해 이렇게 한다

필드를 직접 접근하여 사용 
- 상수는 기능이 아닌 고정된 값 자체를 사용하는 목적으로 만들어진것이다
- 상수는 값을 절대 변경할 수 없다 따라서 필드에 직접 접근하여도 데이터가 변하는 문제가 발생하지 않는다.


음 만약 요구사항중에 모든 차량의 년식은 2011-09-01 이란 요구사항이 붙었다고 해보자

그렇다면 상수로 관리하는 클래스를 만들어

static final String CAR_MAKE_YEARS = 2011-09-01 

이런식으로 상수를 만들어주고 필요할때마다 클래스에서 꺼내와 사용하면 좋을것같다.

매직넘버 x 상수화를 이용하자 ( 유지보수 및 코드 이해도 증가 )


참조 대상 final

참조대상은 바꿀수없지만 참조대상의 변수 값은 바꿀수있다.
